装饰器

四、常见应用场景

方法装饰器

  • 输入验证：检查方法参数合法性。
  • 日志记录：记录方法的调用和参数。
  • 权限控制：验证用户权限后执行方法。
  • 缓存结果：缓存方法返回值（如 @lru_cache）。

类装饰器

  • 注册类：将类添加到全局注册表（如 ORM 模型注册）。
  • 单例模式：确保类只有一个实例。
  • 动态注入方法：为类添加通用方法。
  • 元编程控制：修改类的初始化行为或属性访问。

注意事项

  1. 装饰器顺序：

  2. 继承与装饰器：

    • 类装饰器会影响所有子类，除非子类显式覆盖装饰行为。

  3. 性能影响：

    • 复杂的装饰器可能增加代码执行时间（如频繁的日志记录或输入验证）。

通过灵活使用类装饰器和方法装饰器，可以显著提升代码的可维护性和复用性，但需合理设计以避免过度抽象。
